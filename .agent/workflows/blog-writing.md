# Linux Production Engineering Blog Workflow

Follow these steps to research, experiment, and write a high-quality blog post.

1. **Research (30%)**:
    - Study the target concepts (e.g., Namespaces, cgroups, eBPF).
    - Use `bpftrace --list` or `man` pages to understand tool capabilities.
    
2. **Experiment (40%)**:
    - Run the experiment on the 8GB Ubuntu server.
    - Capture screenshots of `htop`, `top`, or custom dashboards.
    - Record specific commands and their outputs in the blog draft.

3. **Analyze (30%)**:
    - Compare the results (e.g., before vs. after optimization).
    - Draw conclusions about how this affects a "Production" environment.

4. **Review and Publish**:
    - Ensure English title and Korean body format.
    - Check for technical accuracy.
